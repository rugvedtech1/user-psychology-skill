# User Psychology Skill - Auto Activation Instructions

## IMPORTANT: Read This Before Any UI/UX Task

You are now equipped with the User Psychology Skill. This skill must be
automatically applied whenever the user asks you to build, design, create,
or improve any UI, UX, webpage, landing page, form, pricing page, or
any user-facing interface.

## Auto-Activation Rules

Activate this skill automatically when the user mentions any of these:
- Landing page, homepage, website
- UI, UX, interface, design
- Pricing page, checkout, payment
- Signup form, login, onboarding
- Product page, listing, catalog
- Dashboard, admin panel, portal
- Mobile app screen, app design
- CTA, button, conversion
- Any industry: ecommerce, SaaS, fintech, healthcare, edtech

## Your Behavior When Skill is Active

### Step 1 - Detect Industry
First identify which industry the product belongs to:
- Ecommerce → apply FOMO heavily, social proof on product pages, urgency
- SaaS → apply pricing psychology, free trial CTA, feature comparison
- Fintech → apply trust signals heavily, minimal FOMO, security badges
- Healthcare → apply trust signals maximally, avoid dark patterns completely
- Edtech → apply social proof, progress psychology, FOMO on cohorts

### Step 2 - Apply Core Principles
Always apply ALL of these unless industry rules say otherwise:
1. CTA Psychology - use action verbs, high contrast colors, strategic placement
2. Color Psychology - match colors to emotional goals of the industry
3. Social Proof - place reviews, numbers, logos at decision points
4. FOMO/Scarcity - add urgency triggers appropriate to the industry
5. Eye Patterns - structure layout following F-pattern or Z-pattern
6. Above The Fold - ensure hook, value prop, and CTA are above fold
7. Pricing Psychology - use anchoring, decoy pricing, highlight middle plan
8. Form Psychology - reduce fields, add progress bars, use micro-copy
9. Trust Signals - place SSL badges, guarantees near conversion points
10. Avoid Dark Patterns - never use manipulative UX that harms user trust

### Step 3 - Explain Your Decisions
After generating any UI/UX output, always include a section called
"Psychology Decisions" that explains:
- Which principles you applied and why
- Which industry rules triggered
- What psychological effect each design choice creates
- What to A/B test first
